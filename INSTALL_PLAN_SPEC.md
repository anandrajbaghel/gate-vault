# INSTALL_PLAN_SPEC.md

# GATE Manager Installation Plan Specification

Version: 1.0
Status: Draft

---

# Purpose

This document defines the Installation Plan generated by the Installation Planner.
The Installation Plan is the single source of truth describing every action that GATE Manager intends to perform on a vault.
The planner never modifies the filesystem.
It only generates the plan.
Other managers execute the plan.

---

# Philosophy

Planning and execution must remain completely separate.

Planner:
Determine what should happen.

Installer:
Execute the plan.

Updater:
Execute the plan.

Restore:
Execute the plan.

Rollback:
Execute the reverse plan.

No manager except the execution layer should modify the vault.

---

# Input

The planner receives:

- Extracted temporary vault
- Local vault
- vault-manifest.json
- vault-index.json
- Local metadata database

---

# Output

The planner returns exactly one Installation Plan.
The Installation Plan completely describes the operation.
Nothing should be inferred later.

---

# Installation Plan Structure

The plan contains:

- Repository Version
- Local Version
- Manifest
- Statistics
- Ordered Actions
- Warnings
- Errors

---

# Action Types

Every file must belong to exactly one action.

## Install

Repository file does not exist locally.

Action:
Copy file into vault.

---

## Update

Repository file changed.
Local file has not been modified.

Action:
Replace file.

---

## Conflict

Repository file changed.

Local file was modified by the user.

Action:
User decision required.

Options:
- Replace
- Keep Local
- Compare

---

## Merge

Shared ownership file.

Examples:
- Error Logbook
- PYQs Pattern Book
    

Action:
Pass to Merge Engine.
Planner never merges.

---

## Archive

Repository removed the file.

Move local file into:
Archive/
Removed in /
Never permanently delete.

---

## Ignore

User-owned file.
Never modify.
Never move.
Never delete.

---

## Skip

Repository and local copies are identical.
No action required.

---

# Ownership

Ownership comes from:
vault-index.json
Never infer ownership from folder names.
Planner executes repository policy.
Ownership Types:
Repository
Shared
User
Special

---

# Action Priority

Execution order is fixed.
Create directories
Archive
Install
Update
Conflict
Merge
Cleanup
Verification

---

# Action Object

Each action contains:
Path
Action Type
Ownership
Reason
Repository Version
Local Version
Requires Confirmation
Merge Strategy
Warnings
Never use anonymous arrays.

---

# Conflict Rules

Conflicts are never resolved automatically.
Planner only reports them.
Execution Manager asks the user.

---

# Archive Rules

Repository files removed in a release must never be deleted.

Move to:
Archive/
Removed in /
Preserve folder hierarchy.

---

# User Files

User-created files are never touched.
Planner ignores them.
They never appear in Install, Update or Archive actions.

---

# Shared Files

Shared ownership files are never overwritten.
Planner creates Merge actions.
Merge Engine decides the final content.

---

# Metadata

Planner records:
Total Files
Install Count
Update Count
Conflict Count
Merge Count
Archive Count
Ignored Count
Skipped Count
Errors
Warnings

---

# Planner Guarantees

Planner must be deterministic.
Running the planner twice without changes must produce identical plans.
Planner must never modify files.
Planner must never ask the user questions.
Planner must never perform merges.
Planner must never resolve conflicts.
Planner only describes work.

---

# Future Compatibility

The Installation Plan must remain usable by:
Installer
Updater
Restore Manager
Rollback Manager
Selective Installer
Preview Mode
Dry Run
Command Line Tools

Future versions should extend the plan without breaking older fields.

---

# Guiding Principle

The Installation Plan is the contract between decision-making and execution.
Once generated, every later manager should be able to execute it without re-evaluating repository policy.
